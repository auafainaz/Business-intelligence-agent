# Rule: Code Mode

## Coding Instructions

1. Implement only the approved phase/scope.
2. Inspect existing files before editing.
3. Check `docs/langgraph-architecture.md` before changing product flow or graph boundaries.
4. Use existing project conventions first.
5. Use JavaScript for frontend.
6. Use clear Python service boundaries for backend.
7. Keep provider-specific logic isolated in services.
8. Keep the live voice path thin.
9. Keep LangGraph behind live voice.
10. Keep public/internal data separate.
11. If a new environment variable is required, update `.env.example` and docs.
12. Do not hardcode secrets.
13. Do not reintroduce external workflow automation unless explicitly requested.

## Deliverables

- changed code/docs
- validation command/results
- short summary
- remaining risks or manual steps
